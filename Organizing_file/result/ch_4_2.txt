##Router

	최근 가장 중요하게 다뤄지는 네트워크 장치

	OSI 3계층에 해당하는 장치(IP주소를 통해 데이터 이동)

	데이터 패킷이 네트워크 사이를 이동할 수 있게 함

	둘 또는 그 이상의 네트워크 연결에 관여

	브로드캐스트 데이터 패킷들을 필터링해 네트워크 보안 향상
	(관리자가 원할 경우 컨트롤 가능)
	
	IP주소를 이용해 데이터를 이동시킬 수 있는 최적의 경로를 탐색
	(라우터가 포함된 네트워크에 연결된 디바이스들의 IP 정보를 담고 있는 라우팅 테이블 사용)

	Static Routing - 데이터를 이동할 때, 특정 경로를 미리 결정하는 방식
	최적 경로 연산이 필요가 없지만,네트워크가 추가됐을 때 관리자가 설정하지 않으면
	추가된 네트워크로는 데이터가 이동할 수 없음(보수적인 네트워크 환경에서 사용)

	Dynamic Routing - Routing Algorithm을 통해서 최적의 경로를 계산함
	특정 경로에 트래픽이 몰려서 데이터가 지연될 때 효율적인 데이터 전송이 가능함
	네트워크가 추가되도 라우터간의 통신을 통해 데이터 이동이 쉽게 가능함

#Gateway

	과거에는 많이 쓰였으나 최근에는 중요도가 상당히 떨어짐

	OSI 3계층 또는 그 이상에 해당하는 장치

	라우터와 비슷한 역할(네트워크 사이를 중간에서 연결)
	라우터는 양쪽 네트워크가 둘 다 IP를 사용하는 경우,
	게이트웨이는 서로 다른 언어를 사용하는 경우

	최근에는 All-IP화 되어가면서 배제됨

##Network Backbones 

	두 네트워크 사이에 데이터를 고속으로 이동하기 위해
	하드웨어,프로토콜 등을 종합적으로 언급

	발신자와 수신자가 서로 다른 네트워크에 위치할 때 주로 사용

	Fault tolerance - 문제가 생기더라도 지속적으로 데이터를 전송할 수 있는 능력이 있어야 함

	Load Balancing - 목적지까지의 경로가 여럿 있어야 하고
		          데이터가 몰릴 경우 분산시킬 수 있어야 함

	FDDI,ATM,Gigabit Ethernet 등이 대표적인 예
