##Technology overview

	LAN은 특별한 목적을 가지고 하드웨어와 소프트웨어를 이용해 구축

	서버는 주로 워크스테이션 이상급의 컴퓨터가 담당, 서버 제공을 위해 운영체제 탑재

	서버 컴퓨터와 LAN 사이에 네트워크 인터페이스 카드가 중재 역할

	유선(케이블) 또는 무선(주파수 대역)을 통해 구성

	허브, 브릿지, 스위치, 라우터, 게이트웨이 등의 연결장치가 있음

##단순한 형태의 LAN의 물리적 구성

	서버----(유선케이블)----연결장치----(유선케이블)----클라이언트

##서버

	서버:랜에 연결되어 있는 컴퓨터로, 네트워크 기능이 지원되는 운영체제가 탑재되어
	 네트워크 기능이 탑재된 클라이언트들에게 서비스를 제공

	up-and-running(전원이 켜져있고 작동중이어야함) 상태여야 서비스 제공이 가능

##랜 서비스

	데이터 저장, 프린팅, 중앙 통제 등이 있음

	하나의 서버에서 여러개의 랜 서비스도 가능함

##클라이언트

	랜에 연결되어있는 컴퓨터로 특정 서비스를 요청하고 이용할수 있게 구성된 객체

	네트워킹 지원이 되는 운영체제가 필수적

##랜 구성

	메인프레임과 터미널 : 

		서버가 모든 역할을 수행, 클라이언트는 입력을 받아 메인프레임에 전달,
		메인프레임이 제공하는 결과물을 터미널을 통해 보여주는 형태로 구성

		과거에 대용량의 메인프레임 컴퓨터를 사용하던 시기에 적용되던 방식

		터미널 자체는 연산이 불가능함(더미)

		메인프레임 업그레이드와 어플리케이션 개발 비용이 비쌈

		PC와 워크스테이션 출시 이후 제한적으로 적용(슈퍼컴퓨터를 사용하는 데이터센터)

		(더미 터미널----프론트엔드 프로세서----메인프레임)

	Peer to Peer(P2P):

		중앙에 큰 연산이 필요하지 않은 랜 구성에서 주로 적용

		여러 PC가 필요에 따라 클라이언트 혹은 서버로 작동될 수 있음

		피어 컴퓨터들은 보통 같은 workgroup으로 구성됨

		
	Client-dominant:

		PC의 연산 능력이 좋아지면서 도입되기 시작한 구성

		클라이언트가 상당 부분의 작업을 진행함

		연산의 결과물을 서버에 저장하는 정도로 역할이 나뉨

	Client/Server LANs

		클라이언트와 서버가 역할 분담을 통해 각자의 기능을 수행

		통상적으로 서버에서 데이터 접근, 저장 / 클라이언트에서 데이터 조작

		서버와 클라이언트 사이의 데이터 이동을 최소화시켜 성능 향상

	분산처리방식

		최근에 적용되는 방식, 데이터 접근과 저장 부분이 processing 부분과 떨어져있음

		processing 과정은 여러 컴퓨터로 나뉘어져 처리됨

		연산에 대한 처리 요구가 빠르게 바뀔 때 대응하기 좋음

		여러 대의 PC를 배치 후, 수요에 따라 서버를 추가로 구축하는 방식

		수요가 줄어들 경우 PC를 이동 배치하는 것도 가능
